# OWASP Top 10 security review

| Категория риска | Место обнаружения | Описание уязвимости | Возможные последствия эксплуатации | Критичность | Исправление |
| --- | --- | --- | --- | --- | --- |
| Broken Access Control | `SecurityConfig` | Публичное правило `permitAll("/api/auth/**")` было слишком широким для будущих auth endpoint. | Случайное раскрытие новых служебных endpoint без аутентификации. | High | Публичными оставлены только `POST /api/auth/register` и `POST /api/auth/login`; Swagger/OpenAPI требует роль `OPERATOR`. |
| Security Misconfiguration | `SecurityConfig`, `vite.config.ts` | Swagger/OpenAPI был публичным; dev server не ограничивал host явно; security headers не были заданы явно. | Утечка структуры API, случайная экспозиция frontend dev server, повышенный риск clickjacking/referrer leakage. | Medium | OpenAPI закрыт ролью `OPERATOR`; добавлены CSP, frame deny, no-referrer, Permissions-Policy; Vite bind на `127.0.0.1` с allowed hosts. |
| Software Supply Chain Failures | `frontend/package.json`, `frontend/package-lock.json` | npm-зависимости использовали semver ranges `^`, что позволяло незаметную смену версий при установке. | Нерепродуцируемые сборки и повышенный риск supply-chain regression. | Medium | Версии frontend-зависимостей зафиксированы точными значениями; lockfile синхронизирован. |
| Cryptographic Failures | `AuthContext.tsx`, `apiClient.ts`, `LoginPage.tsx`, `AppProperties` | Access JWT мог храниться в persistent `localStorage`; срок JWT можно было настроить слишком большим. | Кража токена при XSS давала длительный доступ; увеличенное окно эксплуатации украденного токена. | High | Удалён persistent remember-me; токен хранится только в `sessionStorage`; legacy localStorage token очищается; JWT expiration ограничен 60 минутами. |
| Injection | `InputNormalizationService`, `PaymentExportService` | Export filename allowlist принимал небезопасные basename вроде `"."`, а короткие имена могли ломать temp-file creation. | 500 вместо контролируемого ответа; риск ошибок path handling при будущих изменениях. | Medium | Basename обязан содержать букву/цифру; temp-file prefix нормализуется; итоговый путь проверяется внутри export directory. |
| Insecure Design | `PaymentService` | `FLAGGED` платёж можно было подтвердить до fraud review. | Обход fraud-механизма и перевод средств по подозрительной операции. | High | Подтверждение блокируется, если платёж имеет статус `FLAGGED` или `flagged=true`. |
| Authentication Failures | `AuthService`, `LoginAttemptService`, `AppProperties` | Login endpoint не ограничивал число неудачных попыток. | Online brute-force и credential stuffing. | High | Добавлен in-memory lockout после серии ошибок, конфигурируемые безопасные границы и ответ `429`. |
| Software or Data Integrity Failures | `PaymentService`, `PaymentExportService` | Fraud flag не защищал платёж от подтверждения; export temp path не проверялся явно после создания. | Нарушение целостности платежного процесса; риск небезопасной записи при изменении path logic. | High | Flagged платежи нельзя подтверждать; временный export path проверяется через `startsWith(exportDir)`. |
| Security Logging and Alerting Failures | `AuthEntryPoint`, `AuthService` | 401 на protected resources фиксировался только в application log; throttled login не имел отдельной audit-записи. | Сложнее обнаружить подбор токенов/доступ к protected API. | Medium | `AuthEntryPoint` пишет `ACCESS_DENIED` в `audit_event`; throttled login пишет `LOGIN_FAILED` с metadata `Login throttled`. |
| Mishandling of Exceptional Conditions | `AuthController`, `InputNormalizationService`, `PaymentExportService` | `GET /api/auth/csrf` обращался к `CsrfToken` при отключённом CSRF и мог завершиться `NullPointerException`; короткий export filename мог дать 500. | Нестабильный API и раскрытие ошибок через generic 500. | Medium | CSRF endpoint удалён как несоответствующий stateless Bearer JWT API; короткие export names обрабатываются безопасно. |

Примечание: OWASP Dependency-Check не запускался по требованию.
